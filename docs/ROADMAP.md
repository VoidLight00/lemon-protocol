## Phase 1: MVP

### 목표
> 핵심 기능이 작동하는 최소 제품 출시

### 스프린트 구성

#### Sprint 1: 프로젝트 셋업

**목표**: 개발 환경 구축

**태스크**:
- [ ] Next.js 14 프로젝트 생성
- [ ] TypeScript 설정
- [ ] Tailwind CSS + shadcn/ui 설치
- [ ] Supabase 프로젝트 생성
- [ ] Vercel 연동
- [ ] ESLint + Prettier 설정
- [ ] 폴더 구조 생성

**산출물**:
- 빈 프로젝트 배포 완료
- CI/CD 파이프라인 작동

---

#### Sprint 2: AI 챗봇 코어

**목표**: 기본 AI 대화 기능 구현

**태스크**:
- [ ] OpenAI API 연동
- [ ] Gemini API 연동
- [ ] API 키 입력 UI
- [ ] 스트리밍 응답 구현
- [ ] 기본 프롬프트 작성
- [ ] 메시지 UI 컴포넌트

**산출물**:
- API 키 입력 → AI 응답 수신 가능

---

#### Sprint 3: 분석 엔진

**목표**: 관계 상황 분석 기능 구현

**태스크**:

- [ ] 프로토콜 생성 프롬프트
- [ ] 응답 포맷터
- [ ] 마크다운 렌더링

**산출물**:
- 상황 입력 → 구조화된 분석 결과

#### Sprint 4: 콘텐츠 라이브러리

- 프로토콜/이론 열람 가능

#### Sprint 5: UI/UX 완성

## 기술 부채 관리

| 주간 | 의존성 보안 업데이트 |
| 월간 | 코드 리뷰 및 리팩토링 |
| 분기 | 성능 프로파일링 |
| 반기 | 아키텍처 리뷰 |
