# 🍋 레몬 프로토콜

> 저에너지 상태 전달 신호

---

## 개요

| 항목 | 내용 |
|------|------|
| **이름** | 레몬 (Lemon) |
| **목적** | 저에너지 상태에서도 연결 유지 |
| **상황** | 바쁘거나 지쳐서 길게 말하기 힘들 때 |
| **기반 이론** | Gottman 비딩 이론, 정서적 노동 |

---

## 문제 상황

### 전형적인 갈등 패턴

```
A: "뭐해?" (연결 요청)
B: (무응답 - 바쁨/지침)
A: "왜 읽씹해?" (오해)
B: "바빴어" (3시간 후)
A: "바빴으면 말이라도 해주지" (서운함 표출)
B: "그럴 에너지도 없었어" (방어)
→ 갈등 에스컬레이션
```

### 핵심 문제

**정서적 노동 비대칭 (Emotional Labor Asymmetry)**

- A의 요구: "상황을 설명해줘"
- B의 상태: "설명할 에너지조차 없어"
- 결과: 커뮤니케이션 교착

## 프로토콜 정의

### "레몬"에 함축된 의미

```
"레몬" = 압축 메시지 패킷
├── "지금 정서적 에너지가 부족해"
├── "길게 설명할 여유가 없어"
├── "너를 무시하는 게 아니야"
├── "나중에 제대로 이야기하자"
└── "이전에 합의한 내용 적용 중"
```

---

## 사용법

### 기본 흐름

```
1. A → B: 연결 요청 ("뭐해?", "게임하자" 등)
          │
          ▼
2. B → A: "레몬" (저에너지 신호)
          │
          ▼
3. A → A: 욕구 명시 ("게임하고 싶었어")
          │
          ▼
4. B → A: 수신 확인 (👍, ❤️, "알겠어" 등)
          │
          ▼
5. [대기]
          │
          ▼
6. B → A: "나 이제 괜찮아!" (여유 발생)
          │
          ▼
7. 소통 재개 → 욕구 충족 논의
```

### 규칙

| 규칙 | 설명 | 이유 |
|------|------|------|
| **레몬 후 욕구 명시 필수** | A는 원하는 바를 짧게 말함 | B가 나중에 뭘 해줘야 할지 알 수 있음 |
| **수신 확인 필수** | B는 최소한 이모지라도 응답 | A가 무시당했다고 느끼지 않음 |
| **비난 금지** | 레몬은 면죄부가 아님 | 도구를 악용하면 신뢰 훼손 |
| **남용 금지** | 매번 레몬만 하면 안 됨 | 의미가 퇴색됨 |

## 대화 예시

### 예시 1: 게임 요청

**Before**:
```
A: 게임하자
B: (읽씹)
A: 왜 안 봐?
B: 바빴어 ㅠㅠ
A: 한마디가 그렇게 힘들어?
B: 너는 내 상황을 몰라서...
→ 싸움
```

**After**:
```
A: 게임하자
B: 레몬
A: 알겠어. 오늘 같이 게임하고 싶었어.
B: 👍
(2시간 후)
B: 나 이제 돼! 하자
A: 좋아!
→ 갈등 없이 연결
```

### 예시 2: 통화 요청

**Before**:
```
A: 통화할 수 있어?
B: (1시간 후) 미안 못 봤어
A: 항상 그러네
B: 뭐가?
→ 과거 언급하며 싸움
```

**After**:
```
A: 통화할 수 있어?
B: 레몬
A: 알겠어. 목소리 듣고 싶었어.
B: 나도 ❤️ 이따 할게
A: 응응
→ 서로 마음 확인
```

## 이론적 배경

### Gottman의 비딩 이론

- **비딩(Bidding)**: 상대방에게 보내는 연결 요청
- **Turn Toward**: 비딩 수용 → 관계 강화
- **Turn Away**: 비딩 무시 → 관계 약화
- 레몬 프로토콜은 Turn Away를 방지하는 최소 응답 시스템

### 정서적 노동

- 감정을 조절하고 표현하는 데 드는 에너지
- 지친 상태에서 "예쁘게 말하기"는 높은 비용
- 레몬 = 정서적 노동 비용 절감 도구
